# 문서 객체 모델 (DOM)

Document Object Model의 약자로, document는 html를 가리킨다. 브라우저는 html 문서를 javascript가 이해할 수 있는 object로 파싱하는데, 이때 생성된 객체를 DOM이라고 한다.

즉, javascript를 사용해서 DOM에 접근하고 이벤트 처리 및 조작하는 것이 DOM의 주목적이다.

css도 동일한 개념으로 파싱하여 CSSOM이 생성된다.

# DOM의 구성 요소

html의 모든 element와 text는 node로 변환되며, 주석 또한 node로 변환된다! 브라우저는 html를 파싱하면서 트리 구조의 node로 변환시킨다. 이를 DOM tree라고 부른다.

모든 node는 서로 부모, 자식 혹은 형제 관계로 연결되어 있다. 예를 들어, html 문서가 아래와 같다면 노드 구조는 다음과 같을 것이다.

<!-- (생략) -->
<body>
  <div id="content">
    <!-- Started! -->
    <p>first</p>
    <p>last</p>
  </div>
</body>
</html>

DOCTYPE: html
└── HTML
    ├── HEAD
    │    └── TITLE
    └── BODY
         └── DIV id="content"
             ├── #comment: Started!
             ├── p
             │   └── #text: first
             └── p
                 └── #text: last

javascript를 통해 위에서 만들었던 html 문서의 <p>에 접근해 본다면..

브라우저의 [1]콘솔 창에 document.querySelectAll('p') 를 입력하면, property와 method를 가지고 있는 객체가 출력된 것을 확인할 수 있다. DOM의 목적인 웹 페이지를 조작하기 위해서는 관련된 property와 method가 필요하다. 이 또한 DOM에서 제공된다.

즉, DOM은 node, property, method를 가지는 객체라고 할 수 있다.

[1] 브라우저의 콘솔 창에서 자바스크립트 언어로 html 문서를 읽을 수 있는 이유도, 결국 브라우저가 DOM을 내장하고 있기 때문.

# Node 객체

모든 DOM 객체는 [2]Node 객체를 상속받는다. 각 Node Level에 맞는 객체를 상속받는데, 위 예제를 예로 들자면 <p>는 HTMLElement, Element, Node, Object 객체로부터 property와 method를 상속받는다.

[2] 모든 객체의 조상 객체는 Object다. 즉, Node와 그 하위 객체의 prototype 종점은 Object prototype이다.

# Node 객체

모든 DOM 객체가 공통으로 사용되는 property와 method를 가지고 있는 객체이다.

# property

  • 구조 관련

    • Node.childNodes: 자식 노드들을 NodeList에 담아 리턴
    • Node.firstChild: 첫번째 자식 노드
    • Node.lastChild: 마지막 자식 노드
    • Node.previousSibling: 이전 형제 노드
    • Node.nextSibling: 다음 형제노드

  • 정보 관련

    • Node.nodeType: 기존에 정의된 상수값 리턴
    • Node.nodeName: Element 명

# method

  • appendchild(child): 부모 노드의 마지막 자식으로 'child' 엘리먼트 추가

  • removeChild(child): 부모 노드의 자식 'child' 엘리먼트 추가

  • innerHTML: HTML 형식의 문자열의 자식 노드 추가 또는 조회
