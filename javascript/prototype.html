# Prototype

자바스크립트를 prototype 기반 언어라고 부르기도 하고 MDN에서도 많이 본 용어기도 하다. Javascript의 this 글을 작성하면서 prototype이 의미하는게 정확히 무엇인지 궁금해졌다.

# prototype 이란

javascript의 모든 객체는 prototype 객체를 가지고 있다. 새로운 객체를 생성한 후 console.dir(xx) 로 확인해보면 prototype이 생성된 것을 확인할 수 있다.

그리고 모든 prototype은 prototype object와 prototype link를 참조하고 있다.

# prototype link

단, arrow func를 통해 생성한 객체에는 prototype이 없다.

우연히 콘솔창에 UDF의 dir를 찍어보다가 arrow function도 함께 찍어봤다.

# 왜 js를 prototype 기반 언어라고 할까

javascript에서는 [1]java의 class 개념이 없다. class 대신 prototype을 이용해서 객체를 생성하고 상속 개념을 가져간다.

[1] ES2015부터 class 문법을 지원하고 있으나 java에서의 class와 동일하진 않음!
